sora_api {sora}R Documentation

Get and set SoRa API

Description

Set a non-default URL to communicate with the SoRa API. This is necessary when working from a secure room or when connecting to a different SoRa service (e.g., from a different country). By default, the sora package sends requests to the default public API, which is not available in secure rooms.

If you are working inside a secure room, make sure to ask the staff for the SoRa URL. You can then set it using sora_set_api(). To retrieve the currently set API, use sora_api().

Usage

sora_api()

sora_set_api(url = NULL, verbose = NULL)

Arguments

url

A valid URL to a SoRa API. If NULL, sets the URL to the default public API of the SoRa service. Defaults to NULL.

verbose

A logical, character string or NULL that specifies if and how to produce messages. If FALSE, suppresses all informative messages. If a path is provided, stores all messages in the specified file. If NULL, verbosity depends on getOption("sora_verbose"). Use the latter to set verbosity across the entire session, e.g. to suppress messages from all package functions. Defaults to NULL and getOption("sora_verbose") defaults to TRUE, i.e., display messages where reasonable.

Details

sora_set_api is only able to set an API URL for the current session. It is reset upon starting a new session. To set a persistent path, set the environment variable manually inside your system's environment variable storage or use an .Renviron file.

Value

Both functions return the currently (or newly) set SoRa API, but sora_set_api returns it invisibly.

Examples

# Get the current API
sora_api()

# Set a new API, e.g., inside an air-gapped network
sora_set_api("ftp://secure-room-data/")
sora_set_api("https://10.1.1.1/")

# Get the new API
sora_api()

# Reset to the public API
sora_set_api()
