sora_datapicker {sora}R Documentation

Data picker

Description

Download and read the information from the SoRa data picker.

This function provides a programmatic interface to the web version of the data picker, which can be found under https://sora.gesis.org/unofficial/datapicker/.

sora_dp_overview(): Get an overview of the datapicker provided by 'SoRa'. The output is a cross table of the both arguments ('arg_1' and 'arg_2').

sora_dp_get_id(): Get an overview of the possible dataset id which can be selected. The output has this columns: 'title', 'time_frame', 'spatial_resolution' and 'dataset_id'

sora_dp_info_id(): Get info, all columns of the datapicker for the selected 'dataset_id'

Usage

sora_datapicker(content = c("provided", "admin", "spatial", "linking"))

sora_arguments(method, provider = "IOER")

sora_dp_overview(
  data_dp = NULL,
  sort_by = "Raster",
  content_dp = "spatial",
  arg_1 = "title",
  arg_2 = "spatial_resolution"
)

sora_dp_get_id(
  data_dp = NULL,
  indicator,
  col_id = "dataset_id",
  content_dp = "spatial"
)

sora_dp_info_id(
  data_dp = NULL,
  col_id = "dataset_id",
  content_dp = "spatial",
  dataset_id
)

Arguments

content

The type of dataset for which to retrieve details. The following values are allowed:

  • "provided": Metadata of coordinate datasets provided by SoRa through sora_provided

  • "admin": Metadata of administrative area datasets recognized by SoRa through sora_admin

  • "spatial": Metadata of geospatial datasets provided by SoRa through sora_spatial

  • "linking": Metadata of available linking methods that can be used in sora_linking

method

A linking method that is used in sora_request and sora_linking to link a (survey) dataset to a spatial dataset. sora_arguments returns the arguments related to the specified linking method.

provider

ID of a linking service provider to perform the spatial linking. Defaults to "IOER".

data_dp

datapicker

sort_by

if 'arg_2' = 'spatial_resolution' is could be that the sorting is not right, if that the case there is the possibility to sort the columns by the argument 'sort_by'. The value 'Raster' is the default for sorting. It can be depends of the data provider.

content_dp

selected table like "spatial"

arg_1

argument 1. The default is 'title'.

arg_2

argument 2. The default is 'spatial_resolution'.

indicator

is the 'title' output in the 'datapicker'

col_id

argument for the 'dataset_id' can be depends of the data provider. Column with the information about the 'dataset_id' in the datapicker.

dataset_id

Specific id for a dataset.

Value

  • sora_datapicker returns a dataframe containing details about provided coordinate datasets, administrative areas, geospatial datasets, or linking methods depending on the value passed to content. Some common columns include:

    • provider: The provider of the dataset.

    • dataset_id: Dataset ID that is used in sora_provided and sora_spatial.

    • description: Description of the dataset. Used in the provenance record.

    • pid: Persistent identifier of a dataset, usually a DOI.

    • required_permissions: Information about the terms of use required to access the data. These need to be accepted when registering for an API key.

  • sora_arguments returns a dataframe where each row is a argument that can be passed to the method specified in method and each column gives further details about the argument.

Examples


sora_datapicker("admin")

sora_datapicker("spatial")

sora_arguments("nearest_poi")
